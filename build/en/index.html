How Qworum improves application integrations

Qworum integrations are interactive, whereas the currently prevalent application integrations consist of non-interactive data transfers. This makes Qworum ğ˜ğ—µğ—² ğ—²ğ—®ğ˜€ğ—¶ğ—²ğ—¿ ğ—”ğ—£ğ—œ ğ˜ğ—²ğ—°ğ—µğ—»ğ—¼ğ—¹ğ—¼ğ—´ğ˜† to use in many scenarios.

For example, Qworum spares much of the development effort involved in calling remote APIs. In other words, Qworum APIs provide their own UIs so that this UI doesn't need to be implemented by each calling application separately.

Also, Qworum APIs typically require a smaller number of calls compared to other APIs. Think about what it takes to browse a remote file system for example.

In addition, Qworum integrations are easier to set up for the calling applications. For example, developers don't need to obtain an API key. Nor is there a need to sign in the user to the remote service, as Qworum APIs will take care of authenticating the user themselves.
