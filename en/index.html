Qworum — The Service Web

Qworum is:

  • the world's first platform for distributed Web applications. Qworum defines a module system for Web applications, where modules may be located anywhere on the Web. In other words, Qworum is a micro-frontend platform that works at Web scale.
  • an advanced Web browser feature that websites can use.
  • a Web browser extension .
  • a technology that radically improves Web developer productivity through modularity and third-party full-page interactive services. Build your Web application by mixing and matching remote or local services and cut development time.
  • the Service Web that complements the HTML-based Document Web and the RDF-based Semantic Web.
  • an upgrade to the World Wide Web to make it provide better support for Web applications. This is needed because the Web was initially conceived as a medium for content, not applications.

Qworum defines:

  • a new top-level Web format called the Qworum script. These contain executable code rather than content as HTML does. Web browsers can receive Qworum scripts from Web servers in an XML document. Web pages can also generate Qworum scripts using JavaScript.
  • an interpreter for Qworum scripts. This client-side software can be implemented by Web browsers natively or as a browser extension.
  • a new client-side storage for storing the execution state of each browser tab. This storage can use IndexedDB behind the scenes.

Qworum is suitable for all Web application frontends, from Jamstack-style static frontends that are executed on the client to traditional dynamic frontends that are generated on the server for each user.

How to enable Qworum for your website

In order to enable Qworum for your website, simply send us an email by using this link. Be sure to set the URL in the subject line to your own website URL. You can leave the email body empty, we don't read it.

If you don't enable Qworum for your website, you will get JavaScript run-time errors on your website if you try to use Qworum.

Enabling Qworum for your website is FREE and always will be.

Specification

You can either directly jump to the Qworum specification, or first read the following in order to get some context about what Qworum is trying to achieve.

A platform for distributed web applications

There are currently two different Webs that exist on the World Wide Web:

  • The Document Web: This the Web that most Web developers will be familiar with. The primary type of content is the HTML page, and the primary content consumers are human users rather than machines.
  • The Semantic Web: This the Web that is intended for machine consumption rather than human consumption. The primary type of content is RDF and its various encodings such as JSON-LD and Turtle.

Qworum adds another type of Web to the mix: the Service Web. Qworum is used for providing software rather than content. Qworum services are:

  • Interactive: They can interact with end-users through HTML pages.
  • Composable: They can call other Qworum services, and even themselves. And Qworum services can return data to their callers. A Qworum application, on the other hand, is simply a Qworum service that does not have a caller.

Note that although Qworum is distinct from the two previous Webs, it builds on them:

  • User interaction occurs through HTML pages. End-users will barely notice that they are using Qworum rather than the Document Web.
  • Qworum services can return semantic data to their callers, in addition to conventional data such as XML-wrapped JSON or any other XML data.

Use cases

The main use cases for Qworum are:

  • Application integration: Different Web-based applications can provide Qworum services to each other and provide a seamless user experience that combines both applications.
  • Productivity gains when developing web applications: An interactive service needs to be implemented and made available on the Web only once. All Web applications can then leverage that service and spare countless development hours.

Qworum is widely applicable. If an online service is interactive, then it can benefit from using Qworum. Here is a foretaste of the types of services that Qworum can enable:

  • User authentication
  • Payment processing
  • Collection management — shopping cart, addressbook/contacts, movies, files, customer information, supplier information, …

You may ask yourself: "Web applications can already use third-party services through remote API calls, how is Qworum better?". The answer is that currently third-party services can only provide business logic through their APIs (whether they are REST-based or not), whereas Qworum services can also bring the UI that goes along with the business logic. This lightens the workload of Web application developers.

Then you may ask yourself: "But aren't OpenID Connect-based services also interactive?". Indeed they are, but OpenID Connect is only applicable to a particular problem domain, whereas Qworum's interactivity concept applies to all Web-based services.

Deliverables

Deliverables that are in late-alpha / pre-beta stage:

  • The Qworum Specification.
  • Browser extension: Only Google Chrome is supported at this time, although adding support for all major browsers is on our roadmap.
  • Client library in JavaScript: This library is intended for use in the Web pages of Qworum applications and services. ⚠️ For this library to work correctly, your website's origin must be explicitly supported by the Qworum browser extension. This is because this library communicates with the Qworum extension.
  • Server library in JavaScript: Suitable for Node.js (and also Deno) applications. Used for sending dynamic Qworum scripts to clients, and for parsing phase parameters received from clients.
  • Server library in TypeScript: Suitable for Deno applications. Used for sending dynamic Qworum scripts to clients, and for parsing phase parameters received from clients.
  • Visual Studio Code extension: Provides XML snippets for writing static Qworum scripts. This VS Code extension is automatically activated for files that have the .qrm.xml extension.
  • Demo: Qworum application (e-commerce site) & 2 Qworum services (shopping cart, payment gateway).

Other planned deliverables:

  • Chrome DevTools extension: For debugging Qworum applications and services.
  • Netlify build plugin: Aims to ensure that Qworum applications and services will run without runtime errors. It does this by checking that the required Qworum resources are available for the website to use.
